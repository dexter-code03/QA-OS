4. 
Introduction
4.1. Purpose
The purpose of this document is to define the functional and technical requirements for integrating the Melissa Data Global Express Entry Cloud API and the Melissa Global Address Verification Cloud API in onboarding . The integration uses Global Express Entry (GEE) for real-time address search/autocomplete (Free Form and Fielded Input) and Global Address Verification (GAV) to verify, correct, and standardize the selected or manually entered address, with optional latitude/longitude output.
4.2. Scope 
Implementation of a client for the RESTful Cloud APIs.Integration of the primary /GlobalExpressFreeForm endpoint and auxiliary endpoints.
Integration of Global Address Verification endpoint /V3/WEB/GlobalAddress/doGlobalAddress for post-selection verification and standardization.Implementation of the secure Token Server for public-facing web pages for GEE.Handling of real-time response/result codes (XS01, XS02, XS03 for GEE; AV/AE/AC/GS codes for GAV).

5.2. Functional Flow (End-to-End User Experience)
Figma flow: 
Figma link
Draw.io : 
Userflow
User completes phone verification (OTP) and enters Full Name and Date of Birth.
The next screen prompts the user to enter their address using a single address search input field.
As the user types in the address search field, the system monitors the input and waits until:
At least ~4 words are entered, 
OR
Approximately 50% of the expected address length is reached.
Once the threshold is met and the user pauses typing (debounce logic applied), the system triggers the Melissa Global Express Entry API.
Debounce logic prevents excessive API calls.
Typical API response time is ~50–150 ms.
The API returns a list of real-time matched address suggestions.
Suggestions are displayed in a dropdown below the input field.
The user can select a suggested address or continue typing/editing manually.
When an address is selected or entered, it is mapped into individual address fields shown to the user.
Allowed editable fields include street name, house/building number, unit/apartment, and neighborhood.
The user reviews the mapped address, edits allowed fields if required, and taps 
Continue
.
The system evaluates the Melissa ResultCode for address verification.
If the ResultCode is 
AV23 or higher
 (e.g., AV23, AV24, AV25) [Back Office configured]:
The address is considered verified.
The user is allowed to proceed to the next step.
If the ResultCode is 
below AV23
 (e.g., AV21, AV22):
The user is shown a friendly error message:
“The address you entered could not be verified. Please try again.”
The user is returned to the address entry/edit screen.
If the user edits one or more allowed fields and the updated address is not found in Melissa records or does not meet the verification threshold:
A confirmation screen is shown to the user.
The confirmation screen provides two options:
Proceed with the address entered/edited by the user.
Select and proceed with the address recommended by Melissa.
The user can choose either option and continue or go back to edit the address again.
Once the user confirms an address, it is saved and the user proceeds to the next onboarding step.
